Predictor Dockerfile templates

Reference / template Dockerfiles distilled from the near-identical images across the Predictor repos. These are not a built package - copy a template into a new service, fill the {{PLACEHOLDER}} tokens, and delete the # --- OPTIONAL --- ... # --- END --- blocks that do not apply.

The three canonical shapes

Shape Template Real-world sources Defining traits
A. Node service node-service.Dockerfile.template heartbeat-service/Dockerfile, faucet/packages/backend/Dockerfile Standalone build context (one package). Two stages: deps (pnpm install --prod) + run. Runs .ts directly via node --experimental-strip-types - no compile step, no lib/. Runs as USER node on port 8080.
B. Subsquid processor subsquid-processor.Dockerfile.template block-explorer/packages/{account,explorer,stats}/Dockerfile, predictor-squids/packages/{node-manager,prediction-markets}/Dockerfile Built from the monorepo root. Private-scope @predictor-foundation auth via a BuildKit --secret id=github_token (never in a layer). Compiles to lib/, packs with pnpm deploy --prod --legacy /deployed. Processor runs lib/main.js (Prometheus 3000); GraphQL query node on 4000.
C. Static SPA + nginx static-spa-nginx.Dockerfile.template, static-spa-nginx.conf.template, docker-entrypoint.sh.template faucet/packages/frontend/Dockerfile, prdctr-bridge-dapp/Dockerfile Compiled SPA served by nginx:1.27-alpine on unprivileged port 8080 with SPA history fallback. Optionally a runtime config.js generated from env at container start.

What varies per repo (the placeholders and optional blocks)

Shape A - Node service

  • Base tag: both use node:24-slim.
  • papi descriptor regeneration (faucet backend only): the faucet copies .papi/ and runs pnpm exec papi to rebuild the gitignored @polkadot-api/descriptors from committed metadata, offline. Heartbeat has no such step. This is the optional block in the template - see also ../papi-descriptors.html for a proposal to share these descriptors instead of regenerating per image.
  • Lockfile: heartbeat copies pnpm-lock.yaml and uses --frozen-lockfile; the faucet backend installs without a per-package lockfile.

Shape B - Subsquid processor

  • Production-stage layout - the one real structural fork:
    • block-explorer: a single squid stage that EXPOSE 3000 4000; the k8s deployment overrides the command to pick processor vs query node.
    • predictor-squids: two stages, processor (--target processor, port 3000) and query-node (npx squid-graphql-server --subscriptions, port 4000).
  • RDS TLS (block-explorer): downloads global-bundle.pem and sets the DB_SSL* / PGSSLMODE env. predictor-squids keeps only NODE_EXTRA_CA_CERTS and no forced DB SSL.
  • Shared workspace package (block-explorer): packages/shared (its package.json, source, and scripts/init-db.js). predictor-squids has no shared package but copies tsconfig.base.json.
  • pnpm version: block-explorer installs latest (npm i -g pnpm); predictor-squids pins pnpm@10.29.2.
  • Per-package tokens: {{PKG_DIR}}, {{PKG_FILTER}} (e.g. @prd/explorer-squid vs @predictor-squids/node-manager), {{CHAIN}} (predictor), and NODE_MAX_HEAP (explorer sets 2048 and passes --max-old-space-size).
  • Install filter: predictor-squids narrows the install with --filter <pkg>...; block-explorer installs the whole workspace (with a --frozen-lockfile || install fallback and --dangerously-allow-all-builds).

Shape C - Static SPA + nginx

  • Build strategy - the main fork:
    • faucet frontend: dist/ is built on CI before docker build; the image is nginx-only (COPY dist ...), no node stage.
    • bridge dapp: a node:22-alpine build stage runs pnpm build, then COPY --from=build /app/dist.
  • Runtime config.js (bridge dapp): a docker-entrypoint.sh writes window.PREDICTOR_BRIDGE_CONFIG from env at start, dropped into /docker-entrypoint.d/ with --chmod=0755 (nginx's runner skips non-executable .sh). The faucet frontend has no runtime config; it proxies /api and /healthz to faucet-backend:8080 instead.
  • Asset caching (bridge dapp): long-cache immutable /assets/, no-store /config.js.

Cross-cutting conventions (shared by all shapes)

  • Unprivileged listen port 8080 for HTTP services (node security group only allows cross-node pod traffic on 1025-65535).
  • Run as USER node in the runtime stage.
  • Private-scope installs authenticate through .npmrc's ${GITHUB_TOKEN} placeholder, supplied as a BuildKit secret - never baked into a layer.
  • pnpm via corepack (node services / SPA) or npm i -g pnpm (squids).
